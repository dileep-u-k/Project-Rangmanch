# 02 — Testing Methodology and Rounds Tracker

## Purpose of this document
This document defines how Project RANGMANCH should be evaluated during Phase 7.

Its purpose is to make testing:
- deliberate
- repeatable
- structured
- and useful for refinement

---

## Core principle
Testing should not be a single vague “it feels good” pass.

The product should be evaluated in rounds, where each round checks a different truth:
- comprehension
- delight
- output quality
- repetition
- continuity
- demo safety

---

# Suggested test rounds

## Round 1 — Internal concept walkthrough
### Goal
Check whether the product is understandable at a glance.

### What to test
- onboarding impression
- home screen clarity
- understanding of Today’s Episode
- understanding of Squad Mode
- understanding of artifact and continuity

### Outputs
- comprehension observations
- confusing terms
- hesitation notes

---

## Round 2 — Prototype flow testing
### Goal
Check whether the main flows work smoothly.

### What to test
- onboarding
- episode setup and completion
- squad setup and completion
- artifact route
- next act route
- archive route

### Outputs
- flow friction points
- route confusion
- broken-feeling transitions
- places where delight weakens

---

## Round 3 — Output quality stress testing
### Goal
Check whether generated or seeded outputs are consistently strong enough.

### What to test
- multiple episode inputs
- multiple squad inputs
- multiple artifact outcomes
- multiple next-act states

### Outputs
- strong output patterns
- weak output patterns
- generic-feeling cases
- artifact-worthy vs weak cases

---

## Round 4 — Demo rehearsal testing
### Goal
Check whether the live presentation path is stable and smooth.

### What to test
- full Flow A
- full Flow B
- full Flow C
- use-demo-example shortcuts
- fallback-safe behavior

### Outputs
- unstable moments
- weak demo timing
- awkward route changes
- screens that need stronger narration

---

## Round 5 — Final acceptance testing
### Goal
Check whether the product is now safe to present.

### What to test
- clarity
- delight
- continuity visibility
- shareability
- demo repeatability
- overall confidence level

### Outputs
- final acceptance decision
- remaining risks
- signoff readiness

---

# Tracking template

For each round, capture:
- test round name
- date
- flow tested
- what worked
- what felt weak
- severity
- recommended action
- current status

---

# Final summary
Phase 7 testing should move from:
- understanding
to
- flow quality
to
- output quality
to
- demo safety
to
- final signoff

That is the correct validation progression.
