# 00 — Phase 3 Overview

## Phase title
**Phase 3 — AI Intelligence Design and Companion System Design**

## Core objective
The purpose of Phase 3 is to define the brain and behavioral soul of RANGMANCH.

This phase answers:
- how the AI companion behaves
- how context is interpreted
- how life moments become entertainment
- how memory works
- how continuity is preserved
- how outputs are structured
- how repetition is controlled
- how tone is governed
- and how the system feels like a living companion rather than a random generator

## Why this phase exists
Phase 2 defined:
- what the product is
- what the main flows are
- what the MVP is
- what the AI must output

Phase 3 now defines:
- how the intelligence decides
- how the companion changes state
- how framing is chosen
- how continuity is injected
- and how the whole experience stays fresh, coherent, and safe

## Core mandate
By the end of this phase, the project must be able to answer clearly:

**When a user opens the app, how does the companion know what to do, what tone to adopt, what kind of entertainment to generate, what to remember, and how to make the experience feel both fresh and continuous?**

## Required outputs
1. Intelligence Architecture Document
2. Companion Persona Spec
3. Mood and Situation Routing Matrix
4. Episode Generation Framework
5. Squad Dynamics Engine Spec
6. Memory Architecture Spec
7. Prompt Specification Handbook
8. Output Schema and Generation Contracts
9. Safety and Tone Governance Note
10. Repetition Control Plan
11. Content Generation Matrix
12. Success Criteria and Failure Modes
13. Exit Gate
14. Signoff

## Code implementation layer
This phase also introduces the `ai-core/` package to encode:
- system contracts
- routing structures
- schemas
- safety policy
- prompt interfaces
- memory models
- freshness logic

## Phase 3 lock rule
By the end of this phase, the intelligence system should be fully understandable without needing live model integration yet.
