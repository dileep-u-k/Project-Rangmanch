# 02 — Information Architecture Map

## Purpose of this document
This document defines the information architecture of Project RANGMANCH.

Its purpose is to specify:
- the top-level structure of the app
- the hierarchy of primary surfaces
- the MVP-active navigation layers
- and how users conceptually move through the world

This is the structural UX map of the prototype.

---

## Core principle
RANGMANCH should not feel like:
- a generic utility app
- a prompt box
- a chatbot with tabs
- or a content feed

It should feel like:
- a living entertainment universe
- with a clear home surface
- strong mode entry points
- and continuity-aware navigation

---

# Top-level information architecture

The full conceptual app structure is:

1. Onboarding
2. Home / Universe
3. Today’s Episode
4. Squad Mode
5. Artifact / Share
6. Continuity / Next Act
7. Archive / Memory
8. Profile / Companion Styling

---

# MVP-active primary IA

For the case-study MVP, the active information architecture should prioritize:

- Home
- Episode
- Squad
- Next Act / Archive light surface

This keeps the prototype focused and easy to demo.

---

# IA hierarchy

## Layer 1 — Entry
- Intro splash / onboarding
- World entry
- First-use setup

## Layer 2 — Primary hub
- Home / Universe
- Companion greeting
- Main CTAs
- Continuity signal

## Layer 3 — Core experiences
- Today’s Episode flow
- Squad Mode flow

## Layer 4 — Outcome and portability
- Artifact / Share surface

## Layer 5 — Continuity and memory
- Next Act
- Archive / Memory

## Layer 6 — Optional personalization
- Profile / Companion styling

---

# IA design rules

## Rule 1
The Home / Universe surface must act as the emotional and structural center of the app.

## Rule 2
The MVP should minimize top-level complexity.

## Rule 3
The Episode and Squad flows should feel like distinct experiences, not just different prompts.

## Rule 4
Artifact and continuity should remain visible in the overall architecture, not hidden.

## Rule 5
Archive and profile should be lighter-priority than the core wow flows.

---

# Primary navigation strategy for MVP

The MVP should use a simple navigation model centered on:

- Home
- Episode
- Squad
- Next Act / Archive

This can be implemented through:
- a bottom navigation bar later
or
- route-based transitions in the prototype shell now

For the prototype, route simplicity is more important than final production nav complexity.

---

# Demo-first IA interpretation

The case-study demo should emphasize these IA paths:

## Demo Flow A
Onboarding -> Home -> Today’s Episode -> Artifact -> Next Act

## Demo Flow B
Home -> Squad Mode -> Squad Result -> Artifact

## Demo Flow C
Home / Next Act -> continuity proof -> Archive light proof

These three flows are enough to prove the product clearly.

---

# Final summary
The information architecture of RANGMANCH is intentionally simple at the top level and rich in the experience layer.

The prototype should therefore feel:
- focused
- emotionally centered on Home
- and clearly oriented around Episode, Squad, Artifact, and Continuity.
