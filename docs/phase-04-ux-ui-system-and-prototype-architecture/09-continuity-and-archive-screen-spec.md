# 09 — Continuity and Archive Screen Spec

## Purpose of this document
This document defines the Continuity / Next Act and Archive / Memory screens of Project RANGMANCH.

Its purpose is to specify:
- how continuity is shown
- how return motivation is expressed
- how recent memory is visualized
- and how the prototype proves that the universe is alive over time

---

## Why these screens matter
Without visible continuity, RANGMANCH risks feeling like:
- a novelty generator
- a one-off demo
- or a sequence of unrelated outputs

These screens prove:
- world continuity
- active hooks
- memory traces
- season/aliveness
- return logic

---

# Continuity / Next Act screen

## Purpose
Make the next-act logic explicit and emotionally legible.

## Must include
- active hook headline
- hook explanation
- why the user should care
- CTA to continue
- CTA to home or archive

## Design goals
- anticipation
- continuity
- momentum
- low pressure
- product aliveness

---

# Archive / Memory screen

## Purpose
Show that prior moments persist as a lightweight world history.

## Must include
- prior episode trace
- prior squad trace
- prior artifact trace
- active or recent continuity signal
- optional “season” framing

## Design goals
- proof of continuity
- lightweight nostalgia
- compactness
- visual variety
- clear memory hierarchy

---

# Interaction guidance
The user should be able to:
- revisit a recent moment
- see what the current world remembers
- understand that the world has not reset
- move easily back to Home

---

# MVP implementation guidance
For the MVP prototype:
- continuity can be lightly mocked
- archive can show a small set of seeded traces
- a full memory engine is not required yet

But the screens must still prove:
- continuity exists
- memory exists
- the world feels alive

---

## Final summary
The Continuity and Archive screens are not optional fluff.
They are core proof surfaces for:
- return logic
- continuity
- memory
- and living-universe feel.
