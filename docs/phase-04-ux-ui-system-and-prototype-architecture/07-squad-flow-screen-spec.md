# 07 — Squad Flow Screen Spec

## Purpose of this document
This document defines the Squad flow of Project RANGMANCH.

Its purpose is to specify:
- how the user enters group context
- how social energy is framed
- how the squad result is shown
- how verdicts and roles are rendered
- and how the flow transitions into artifact and continuity

---

## Why this flow matters
Squad Mode is the flagship social wedge of the MVP.

If this flow is weak, the product loses:
- social spread plausibility
- group recognizability
- screenshot-worthiness
- differentiated companion behavior

---

# Squad flow structure

The Squad flow should be:

1. Squad Setup
2. Squad Result
3. Artifact handoff
4. Optional continuity tease

---

# Screen 1 — Squad Setup

## Purpose
Capture group context with low friction.

## Must include
- friend labels or names
- optional vibe selection
- quick CTA

## Design goals
- compact
- playful
- low typing burden
- obvious move into result

---

# Screen 2 — Squad Result

## Purpose
Render the social entertainment output.

## Must include
- squad title
- squad frame
- member roles
- verdict or ranking
- social payoff line
- CTA to artifact

## Design goals
- instantly readable
- funny or sharply observant
- compact
- screenshot-friendly

---

# Interaction design rules

## Rule 1
Squad setup should not feel like data entry.

## Rule 2
The result should foreground recognizability, not complexity.

## Rule 3
The verdict should feel structured, not random.

## Rule 4
The share path should feel obvious.

---

# MVP implementation note
Mock squad data is acceptable at this stage as long as:
- role structure is clear
- verdict is clear
- artifact handoff is clear
- UI communicates strong social energy

---

## Final summary
The Squad flow should feel like:
- quickly naming the current group chaos
- watching the companion assign roles and judgment
- and getting something that feels worth sharing.
