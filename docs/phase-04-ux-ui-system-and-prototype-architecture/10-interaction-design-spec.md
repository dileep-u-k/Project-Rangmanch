# 10 — Interaction Design Spec

## Purpose of this document
This document defines the interaction system of Project RANGMANCH.

Its purpose is to specify:
- how the user interacts with the app
- what recurring interaction patterns exist
- how interaction stays mobile-natural
- and how the product avoids feeling like a text-heavy AI tool

---

## Core principle
RANGMANCH should feel:
- lightweight
- playful
- card-driven
- tap-friendly
- and dramatically staged

It should not feel:
- form-heavy
- chat-heavy
- typing-heavy
- or workflow-like

---

# Primary interaction patterns

## Pattern 1 — Quick-select chips
Used for:
- moods
- vibes
- humor style
- squad energy

Purpose:
- reduce typing
- increase speed
- preserve delight

---

## Pattern 2 — Mode cards
Used for:
- Today’s Episode
- Squad Mode
- Continuity / Next Act entry

Purpose:
- make the main actions obvious
- create strong emotional hierarchy
- communicate product modes visually

---

## Pattern 3 — Branching choice buttons
Used in:
- Episode Experience

Purpose:
- provide lightweight agency
- make interaction feel meaningful
- avoid long conversational branching

---

## Pattern 4 — Outcome CTA row
Used in:
- Episode Outcome
- Squad Result
- Artifact
- Next Act

Purpose:
- move users smoothly into:
  - artifact
  - home
  - continuity
  - archive

---

## Pattern 5 — Continuity cards
Used in:
- Home
- Next Act
- Archive

Purpose:
- make the world feel alive
- expose active hooks clearly
- keep return logic visible

---

# Interaction rules

## Rule 1
Primary actions should always be obvious.

## Rule 2
Typing should be minimized in MVP.

## Rule 3
A screen should usually have one dominant action and one or two secondary ones.

## Rule 4
The product should reward taps with progression, not dead ends.

## Rule 5
Interaction should feel like:
- entering an act
- choosing a version
- seeing the payoff
- and moving into the aftermath

---

# Interaction speed guidance

The MVP should optimize for:
- low-friction inputs
- clear progression
- fast emotional payoff
- minimal confusion

This means:
- fewer fields
- clearer chips
- stronger card hierarchy
- simpler CTA rows

---

# Screen-specific interaction guidance

## Onboarding
- one text field max
- mostly chip-based
- one strong enter CTA

## Home
- obvious top two cards
- quick mood chips
- continuation card with direct CTA

## Episode
- one short setup field
- clear generate button
- clear choice cards
- clear artifact / next act handoff

## Squad
- a few light friend entries
- one vibe selector
- one generate CTA
- one artifact CTA

## Artifact
- save/share/home/next-act actions

## Next Act
- continue / archive / home actions

---

## Final summary
The RANGMANCH interaction system should feel:
- fast
- playful
- staged
- and mobile-native

Its job is to help the user move from:
- moment
to
- entertainment
to
- payoff
to
- continuity
with almost no friction.
