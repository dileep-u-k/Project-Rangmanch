# 11 — Visual Language and Design System Note

## Purpose of this document
This document defines the visual language of Project RANGMANCH.

Its purpose is to specify:
- the world aesthetic
- type hierarchy
- card hierarchy
- color logic
- and motion style

This is the note that protects the prototype from looking generic.

---

## Core visual principle
RANGMANCH should feel like:
- performance
- stagecraft
- dramatic motion
- entertainment energy
- living universe

It should not feel like:
- a productivity tool
- a dashboard
- a plain chat client
- or a hackathon wrapper

---

# World aesthetic

The world should feel:
- dark-stage
- cinematic
- vivid in accent
- slightly theatrical
- premium but playful

Recommended emotional palette:
- deep midnight backgrounds
- rich indigo / violet surfaces
- vivid magenta-lilac accents
- bright readable text

---

# Typography hierarchy

## Layer 1 — Act titles / hero titles
Use:
- strong serif or cinematic display style

Purpose:
- episode titles
- artifact headlines
- hook headlines

## Layer 2 — Product headings
Use:
- strong modern sans

Purpose:
- section headers
- mode card titles
- screen headers

## Layer 3 — Body/support text
Use:
- clear readable sans
- compact but premium

Purpose:
- descriptions
- continuation lines
- input support copy

---

# Card system

The prototype should use reusable card families:

## Card 1 — Companion banner
Purpose:
- emotional framing
- world entry

## Card 2 — Mode card
Purpose:
- primary actions
- mode distinctions

## Card 3 — Episode card
Purpose:
- structured act framing

## Card 4 — Squad card
Purpose:
- social result framing

## Card 5 — Hook card
Purpose:
- continuity and anticipation

## Card 6 — Artifact frame
Purpose:
- premium portable output

---

# Color and mood logic

The system may later vary mood color accents, but the MVP should already imply:

- chaos -> electric purple / magenta edge
- drama -> deep violet / rich contrast
- chill -> cooler softer dark blue
- celebration -> brighter warmer accent
- continuity -> luminous but controlled accent

The MVP does not need full dynamic theming, but should feel emotionally alive.

---

# Motion language

Motion should imply:
- reveal
- stage shift
- act change
- progression
- next beat

It should not feel:
- flashy for no reason
- chaotic
- over-animated
- or gamey

Recommended motion style:
- subtle fade-ins
- slide-up reveal
- soft scale emphasis on hero surfaces
- compact transitions

---

# UI cleanliness rules

## Rule 1
Screens should not feel overcrowded.

## Rule 2
Cards should carry hierarchy, not just decoration.

## Rule 3
White space is part of the premium feel.

## Rule 4
The world should feel “alive” through composition, not clutter.

## Rule 5
Artifact screens should feel more premium than regular flow screens.

---

## Final summary
The RANGMANCH visual system should make the prototype feel:
- premium
- theatrical
- emotionally expressive
- and unmistakably unlike a generic AI utility.
