# 12 — Companion UI Behavior Note

## Purpose of this document
This document defines how the companion appears in the interface of Project RANGMANCH.

Its purpose is to specify:
- where the companion is visible
- when it speaks directly
- how it frames screens
- and how the prototype avoids becoming a standard chat app

---

## Core principle
The companion should be:
- visible
- emotionally present
- product-defining
- but not the entire interface

The app should not become:
- one long chat thread
- a bubble conversation UI
- or a text-stream product

RANGMANCH should instead use a hybrid interface:
- companion framing
- stage cards
- structured output blocks
- interaction choices
- artifact surfaces

---

# Companion surfaces

## Surface 1 — Hero banner presence
Used in:
- Onboarding
- Home
- sometimes key major screens

Purpose:
- emotional tone setting
- world framing
- “the companion is here” signal

---

## Surface 2 — Contextual narration card
Used in:
- Episode Experience
- Outcome
- Next Act

Purpose:
- frame or punctuate the current act

---

## Surface 3 — Commentary block
Used in:
- Episode Outcome
- Squad Result
- Artifact context

Purpose:
- add personality without taking over the screen

---

## Surface 4 — Continuity line
Used in:
- Home continuity card
- Next Act
- Archive traces

Purpose:
- keep the world alive

---

# Behavior rules

## Rule 1
The companion should usually introduce, frame, or punctuate — not dominate every screen.

## Rule 2
The companion should rarely appear as a full scrolling chat thread in the MVP.

## Rule 3
Companion copy should be:
- concise
- emotionally clear
- slightly theatrical
- branded
- not overly verbose

## Rule 4
The companion should shift posture across contexts:
- onboarding -> host
- home -> host / continuity keeper
- episode -> narrator
- squad -> judge / commentator
- next act -> continuity keeper

---

# MVP UI interpretation
In the prototype, the companion should mainly appear through:
- banners
- narration cards
- commentary sections
- continuity cards

That is enough to prove presence without collapsing into chat UI.

---

## Final summary
The RANGMANCH companion should feel like:
- a visible host
- a narrator of acts
- a judge of chaos
- and a keeper of continuity

But it should not take over the interface as a generic chat bot.
