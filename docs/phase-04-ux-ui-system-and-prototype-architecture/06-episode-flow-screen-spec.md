# 06 — Episode Flow Screen Spec

## Purpose of this document
This document defines the Episode flow of Project RANGMANCH.

Its purpose is to specify:
- the full solo wow flow
- what the user inputs
- how the generated episode is shown
- how choices appear
- how the payoff is revealed
- and how the flow transitions into artifact and continuity

---

## Why this flow matters
Today’s Episode is the flagship MVP experience.

If this flow is weak, the product loses:
- the clearest product proof
- the first wow moment
- the strongest solo delight path

---

# Episode flow structure

The Episode flow should be:

1. Episode Setup
2. Episode Experience
3. Episode Outcome
4. Artifact handoff
5. Next Act handoff

---

# Screen 1 — Episode Setup

## Purpose
Capture the current moment quickly.

## Must include
- mood selection
- short situation input
- low-friction CTA

## Design goals
- minimal typing
- strong clarity
- immediate momentum

---

# Screen 2 — Episode Experience

## Purpose
Render the structured entertainment object.

## Must include
- episode title
- setup
- cast frame
- 2–3 choices

## Design goals
- not chat-like
- visually staged
- easy to understand quickly
- high dramatic clarity

---

# Screen 3 — Episode Outcome

## Purpose
Deliver the payoff.

## Must include
- outcome headline
- outcome summary
- companion commentary
- clear CTA to artifact
- clear CTA to next act

## Design goals
- satisfying
- compact
- artifact-ready
- continuity-aware

---

# Interaction design rules

## Rule 1
The user should never feel lost in the flow.

## Rule 2
Choices should feel distinct and easy to tap.

## Rule 3
The flow should feel like an act progression, not a form followed by a blob of text.

## Rule 4
The payoff should arrive fast enough to preserve mobile attention.

---

# MVP implementation note
The prototype can use mock generated data here as long as:
- the structure matches the intended contracts
- the flow feels real
- the artifact and continuity handoffs are visible
- the UI is convincing

---

## Final summary
The Episode flow should feel like:
- entering a moment
- watching it become staged entertainment
- choosing a path
- getting a payoff
- and being given both a shareable result and a next act.
