# Phase 0 — Strategic Grounding and Thesis Lock

## Purpose
Phase 0 exists to ensure that Project RANGMANCH is not built on ambiguity.

Before any design, code, prototype, or pitch work begins, this phase locks:
- the exact meaning of the product
- the problem it solves
- why this category should exist
- why the idea is different
- what the case brief really requires
- what kind of product is actually being built

## Why this phase is critical
Without Phase 0, the project can drift into:
- generic chatbot behavior
- therapist-first companion framing
- vague differentiation
- feature sprawl
- weak adoption logic
- shallow pitch narrative

## Required deliverables
- Founder-Grade Brief Rewrite
- Thesis Note
- Project Boundary Note
- Category Definition and Originality Note
- Central Metaphor Lock Note
- Product Principles Note
- MVP Wedge Freeze Note
- Success Criteria and Failure Modes
- Exit Gate
- Phase 0 Signoff

## Success definition
Phase 0 is successful only when the following are clear:
1. What is RANGMANCH in one sentence?
2. What exact product category does it define?
3. What makes it different from existing AI companions?
4. What is the first strong wedge?
5. What must the MVP prove?
