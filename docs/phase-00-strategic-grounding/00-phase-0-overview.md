# 00 — Phase 0 Overview

## Phase title
**Phase 0 — Strategic Grounding and Thesis Lock**

## Core objective
The purpose of Phase 0 is to ensure that Project RANGMANCH is not built on confusion.

Before any design, code, prototype, or deck work begins, this phase locks:
- the exact meaning of the product
- the problem it solves
- why this category should exist
- why this idea is different
- what the brief really requires
- and what kind of product is actually being built

## Why this phase exists
Most projects fail because they start too early with:
- features
- UI
- model prompts
- tech stack choices

That produces:
- generic products
- fuzzy differentiation
- weak adoption logic
- and decks that sound polished but lack depth

For RANGMANCH, this phase is essential because the idea is ambitious and category-creating.

## The key questions Phase 0 must answer
- What is RANGMANCH, precisely?
- What is it not?
- What is the non-obvious insight?
- Why is this an AI companion product at all?
- Why is it entertainment-first?
- Why will users care?
- Why is it not a chatbot clone?

## Required outputs of this phase
1. Founder-Grade Brief Rewrite
2. Thesis Note
3. Project Boundary Note
4. Category Definition and Originality Note
5. Central Metaphor Lock Note
6. Product Principles Note
7. MVP Wedge Freeze Note
8. Success Criteria and Failure Modes
9. Exit Gate
10. Phase 0 Signoff

## Phase 0 lock rule
No product research, feature definition, AI system design, UI design, prototype work, or pitch work should proceed unless Phase 0 has been completed and signed off.
