# 01 — Phase 2 Product Definition Plan

## Purpose of this document
This document defines the exact work plan for Phase 2.

Its purpose is to ensure that product definition happens systematically and that the product is frozen clearly before UX design and engineering begin.

## Phase 2 mission
Turn the strategic foundation and research-backed opportunity into a concrete product blueprint.

## What this phase must define

### A. Complete product surfaces
Map all major product surfaces:
- onboarding
- home / universe
- companion interaction layer
- Today’s Episode setup and experience
- episode outcome
- Squad Mode setup and result
- share artifact
- continuity / return hook
- archive / memory
- profile / customization

### B. Core product modes
Define all top-level modes:
- Today’s Episode
- Squad Mode
- Scene This
- Challenge Me
- What Next?
- Live Moment Mode

Then split them into:
- MVP core
- optional MVP teaser
- post-MVP modes

### C. Session architecture
Define the backbone of a session:
1. entry
2. context capture
3. entertainment frame selection
4. interactive experience
5. payoff
6. share moment
7. return hook

### D. User journeys
Define at minimum:
- first-time user journey
- repeat solo journey
- squad / social journey
- continuity / return journey

### E. Output system
Define what the product can output:
- title cards
- episode scenes
- interactive choices
- outcomes
- verdict cards
- squad outputs
- return hooks

### F. Continuity system
Define:
- what carries across sessions
- what stays fresh
- what is remembered
- what becomes part of the user’s evolving “season”

### G. MVP freeze
Lock the exact submission MVP:
- onboarding
- home / universe
- Today’s Episode flow
- Squad Mode flow
- share artifact screen
- return hook / continuity signal

### H. Out-of-scope lock
Explicitly exclude from MVP:
- full social graph
- advanced monetization
- multiple deep event systems
- complete archive engine
- complex analytics infra
- production-scale moderation system
- voice mode
- rich media generation stack

## Working rule
Phase 2 should optimize for:
- clarity
- coherence
- focus
- originality
- shareability
- and retention logic

It should not optimize for adding more features than the product can clearly justify.
