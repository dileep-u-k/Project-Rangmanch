# Phase 2 — Product Definition and Experience Architecture

## Purpose
Phase 2 converts the thesis and research into the actual product blueprint of RANGMANCH.

This phase defines:
- what exactly the product is
- what it does
- what its major surfaces are
- what the top-level modes are
- how a session is structured
- what the user journeys are
- what the MVP includes
- what is out of scope
- and how the whole product becomes coherent before UI polish and engineering

## Why this phase matters
A common mistake is jumping directly from research into wireframes or prototype building.

Before UI and engineering, the product needs:
- architecture
- feature hierarchy
- session logic
- surface definition
- mode structure
- return loops
- and a frozen MVP

Without this, later work becomes messy.

## What this phase must produce
By the end of Phase 2, the project should have:
- a Product Requirements Document (PRD)
- an Experience Architecture Document
- User Journey Maps
- an MVP Freeze Note
- an Output Schema Note
- a Continuity System Concept
- a success criteria file
- an exit gate
- a signoff note
