# 05 — Core Product Modes

## Purpose of this document
This document defines the top-level modes of Project RANGMANCH.

Its purpose is to answer:
- what the major product modes are
- what each mode is for
- how each mode contributes to the overall product system
- which modes are MVP
- which modes are optional
- and which modes are post-MVP

This prevents the product from becoming a vague set of interesting features.

---

## Why mode definition matters
A product like RANGMANCH can easily sprawl because the concept naturally allows many entertaining directions.

Without mode discipline, the product can become:
- unfocused
- overloaded
- unclear to users
- and hard to build well

Mode definition protects:
- clarity
- feature hierarchy
- MVP quality
- future roadmap discipline

---

# Core product mode set

The core mode set of RANGMANCH is:

1. Today’s Episode
2. Squad Mode
3. Scene This
4. Challenge Me
5. What Next?
6. Live Moment Mode

These are the canonical product modes.

---

# Mode 1 — Today’s Episode

## Purpose
This is the flagship solo entertainment mode.

It transforms the user’s current mood, situation, or moment into a personalized interactive entertainment experience.

## Core value
- personal relevance
- fast delight
- structured interaction
- life-to-entertainment transformation

## Typical user input
- mood
- short moment description
- vibe
- quick preset scenario

## Typical output
- episode title
- cast framing
- setup
- 2–3 interaction choices
- outcome
- artifact
- return hook

## Why it matters
This is the clearest proof of the core product thesis.

## MVP status
Required
Primary flagship mode

---

# Mode 2 — Squad Mode

## Purpose
This is the flagship social entertainment mode.

It transforms friend-group dynamics into:
- verdicts
- mini-dramas
- role assignments
- rankings
- or socially legible entertainment objects

## Core value
- group specificity
- social recognition
- shareability
- portable fun

## Typical user input
- friend names
- friend archetypes
- social setup
- current group chaos or energy

## Typical output
- squad title
- roles
- group dynamics framing
- verdict / ranking / scene result
- artifact
- continuation line

## Why it matters
This proves the social expansion logic of the product.

## MVP status
Required
Primary social wedge

---

# Mode 3 — Scene This

## Purpose
Turn a specific moment into a more stylized scene on demand.

This mode is more direct and compositional than Today’s Episode.

## Core value
- sharp transformation
- user-directed framing
- specific scenario dramatization

## Typical user input
- “scene this”
- “turn this into a dramatic scene”
- specific event or moment

## Typical output
- compact dramatic reframing
- stylized scene summary
- possible mini artifact

## Why it matters
This is a natural expansion of the life-to-entertainment thesis.

## MVP status
Post-MVP
Optional teaser only if needed

---

# Mode 4 — Challenge Me

## Purpose
Give the user a playful task, dare, social challenge, or expressive mission framed by the companion.

## Core value
- interactivity
- novelty
- social activation
- energy shift

## Typical user input
- mood
- need for fun
- squad presence or absence

## Typical output
- challenge framing
- task or mission
- optional reward logic
- artifact / result summary

## Why it matters
This could become a strong future retention layer, but it is not necessary to prove the submission MVP.

## MVP status
Post-MVP
Optional teaser only

---

# Mode 5 — What Next?

## Purpose
A continuity-led mode focused on:
- active arcs
- pending aftermath
- companion callbacks
- “continue the world” behavior

## Core value
- continuity
- retention
- world feeling
- next-act exploration

## Typical user input
- re-entry into prior flow
- explicit curiosity about continuation

## Typical output
- continuation framing
- next act
- active thread recap
- possible branch into a new episode or squad scene

## Why it matters
This is one of the strongest long-term expansion layers for retention.

## MVP status
Post-MVP as a full mode
But its logic must be visible in MVP through the Return Hook surface

---

- synchronized reaction
- event amplification
- contextual social fun

## Typical input
- event context

## Typical output

- squad reaction logic
- live verdicts

Optional teaser only

---

# Mode hierarchy

## Tier 1 — Core MVP-active modes
1. Today’s Episode
2. Squad Mode

These are the two main active experience modes in the MVP.

---

## Tier 2 — MVP support layers
3. Share Artifact
4. Return Hook / Continuity Signal

These are not treated here as “modes” in the same sense, but they are mandatory supporting systems of the MVP.

---

## Tier 3 — Optional teaser layers
5. Challenge Me teaser
6. Live Moment Mode teaser
7. Archive / Memory teaser
8. Profile / Style teaser

These may appear lightly if time allows, but are not core MVP requirements.

---

## Tier 4 — Post-MVP expansion modes
9. Scene This
10. Challenge Me full mode
11. What Next? full mode
12. Live Moment Mode full mode

These are part of the future roadmap, not the submission-critical core.

---

# Why Today’s Episode and Squad Mode are the correct MVP modes

## Today’s Episode proves
- life-to-entertainment transformation
- companion framing
- structured interactive entertainment
- personal relevance

## Squad Mode proves
- social relevance
- group identity
- portable fun
- growth plausibility

Together, they are the strongest possible mode pair for the submission MVP.

---

# What should not become a top-level MVP mode yet

The MVP should not try to make all of these top-level active modes:
- Scene This
- Challenge Me
- What Next?
- Live Moment Mode

Why:
- too much choice can reduce clarity
- too many modes dilute polish
- too many modes make the product feel scattered
- the MVP should prove the core thesis, not the full future universe

---

# Mode coherence rules

## Rule 1
Every mode must still feel like RANGMANCH, not like a different app.

## Rule 2
The companion must remain the host, framer, or orchestrator in every mode.

## Rule 3
Modes should differ by experience logic, not by becoming unrelated feature experiments.

## Rule 4
Modes should eventually feed one coherent universe rather than separate silos.

## Rule 5
MVP should focus on mode depth, not mode count.

---

## Final summary
The core mode architecture of RANGMANCH is intentionally broader than the MVP, but the MVP-active product must stay focused.

The product’s long-term mode system includes:
- Today’s Episode
- Squad Mode
- Scene This
- Challenge Me
- What Next?
- Live Moment Mode

But for the submission MVP, the active build set is:

- Today’s Episode
- Squad Mode
- Share Artifact
- Return Hook

That is the correct mode lock for clarity, quality, and proof strength.
