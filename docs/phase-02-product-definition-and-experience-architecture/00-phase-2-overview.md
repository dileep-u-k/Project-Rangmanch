# 00 — Phase 2 Overview

## Phase title
**Phase 2 — Product Definition and Experience Architecture**

## Core objective
Phase 2 converts research and thesis into the actual product blueprint of RANGMANCH.

This phase answers:
- what exactly the product is
- what it does
- what its major surfaces are
- what the user flow is
- what the main modes are
- what the MVP includes
- and how the whole product becomes coherent

## Why this phase exists
Phase 0 defined what RANGMANCH is.
Phase 1 proved why it deserves to exist.

Phase 2 now defines how the product is actually structured before UI polish and engineering begin.

Without this phase, later work risks:
- vague flows
- confusing screens
- mode sprawl
- weak first-session wow
- weak share logic
- weak return logic
- and unstable MVP scope

## Core product questions this phase must answer
- What is the home experience?
- What is the first-session wow?
- What are the top modes?
- How does Today’s Episode work?
- How does Squad Mode work?
- What does the companion do in the product?
- What gets shared?
- Why does the user return tomorrow?
- What is the MVP?
- What is explicitly out of scope for now?

## Required outputs
1. Product Requirements Document (PRD)
2. Experience Architecture Document
3. Product Surfaces and Screen System
4. Core Product Modes
5. Session Architecture
6. User Journey Maps
7. Feature Hierarchy and Scope Map
8. MVP Freeze Note
9. Output Schema Note
10. Continuity System Concept
11. Success Criteria and Failure Modes
12. Exit Gate
13. Signoff

## Phase 2 lock rule
By the end of this phase, the product should be fully understandable without needing any code, mockups, or implementation details.
