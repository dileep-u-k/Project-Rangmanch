# 03 — Core Flow Build Plan

## Purpose of this document
This document defines the build plan for the four core MVP proof flows of Project RANGMANCH.

Its purpose is to ensure that implementation effort stays focused on the strongest user-visible loops.

---

## Core proof flows

### Flow A — Today’s Episode
A user enters a vibe and moment, receives a structured mini-episode, makes a choice, sees an outcome, receives an artifact, and ends with a hook.

### Flow B — Squad Mode
A user enters friend-group context, receives roles and verdicts, and gets a socially portable output.

### Flow C — Artifact
The app turns the result into a clean, branded visual object that feels worth saving or showing.

### Flow D — Return Hook
The app closes with continuity and gives the user a believable reason to return.

---

## Build order

### Step 1
Harden onboarding and home so the product identity is strong before deeper flow work.

### Step 2
Harden Today’s Episode so the strongest wow path works smoothly.

### Step 3
Harden Squad Mode so the social wedge feels alive and screenshot-worthy.

### Step 4
Harden Artifact and Return Hook so the product has portability and continuity.

---

## Flow quality rule
Each core flow should be:
- understandable quickly
- visually clear
- emotionally distinct
- and stable enough for demo repetition

---

## Final summary
The MVP is not “many features.”
It is four well-built proof loops that together make the product feel original, entertaining, portable, and alive.
