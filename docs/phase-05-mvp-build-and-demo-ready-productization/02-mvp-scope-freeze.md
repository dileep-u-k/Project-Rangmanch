# 02 — MVP Scope Freeze

## Purpose of this document
This document freezes the actual build scope of the Phase 5 MVP.

Its purpose is to prevent feature creep and protect the product’s strongest proof loop.

---

## MVP must include

### Core flow A — Today’s Episode
- setup
- generated episode
- branching choice
- outcome
- artifact handoff
- return hook handoff

### Core flow B — Squad Mode
- squad setup
- squad result
- artifact handoff
- optional continuity handoff

### Core flow C — Artifact
- premium artifact frame
- save/share surface
- back-to-home and next-act actions

### Core flow D — Continuity
- visible next-act logic
- archive proof
- home reflection of recent use

### Core product surfaces
- onboarding
- home
- episode
- squad
- artifact
- next act
- archive

---

## MVP may include if time allows
- better archive richness
- stronger dynamic greetings
- more varied seeded outputs
- improved motion polish
- richer empty states

---

## Explicitly out of scope for Phase 5
- full production AI inference backend
- voice mode
- social graph
- multiplayer sync
- rich media generation
- final analytics stack
- deep production memory system
- advanced moderation backend

---

## Final scope rule
If a feature does not strengthen:
- wow
- social proof
- shareability
- continuity
- or demo stability

it should not be prioritized in Phase 5.
